Binding transitions to SceneTransitions

Just as with ViewControllers, Acorn needs to know when to use your SceneTransition implementation, or to fallback to one of the default types. A mapping needs to be created for this as well, and is done in a similar way as with ViewControllers.

The SceneTransitionFactory

The SceneTransitionFactory interface provides a transitionFor(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): SceneTransition function where you can define which implementation to use for a transition. The supports(Scene, Scene, TransitionData?): Boolean method in that same interface allows us to compose several SceneTransitionFactories together.

1 interface SceneTransitionFactory {
2 
3     fun supports(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): Boolean
4     fun transitionFor(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): SceneTransition
5 }

A simple implementation for a transition from MyScene1 to MyScene2 could look like this:

 1 class MySceneTransitionFactory : SceneTransitionFactory {
 2 
 3     override fun supports(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): Boolean {
 4         return previousScene is MyScene1 && newScene is MyScene2
 5     }
 6     
 7     override fun transitionFor(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): SceneTransition {
 8         return MyScene1MyScene2Transition()
 9     }
10 }

TransitionData

The TransitionData parameter can contain useful information about the transition. Currently the class only contains an isBackwards property to denote whether the transition should be regarded as a 'step back in navigation'. The TransitionData parameter can be null if the Navigator that caused the transition does not support the notion of 'backwards', such as the ReplacingNavigator.

1 override fun transitionFor(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): SceneTransition {
2     return when (data?.isBackwards) {
3         true -> BackwardsTransition()
4         else -> ForwardsTransition()
5     }
6 }

Transition extensions

Often screen transitions need some sort of 'default' behavior, such as hiding the keyboard when leaving the screen. Instead of having to write the code for hiding the keyboard in your transition implementation, Acorn provides a special DoBeforeTransition class that can be used to execute a common block of code before the actual transition starts.

The SceneTransition.doOnStart allows you to hook into this class. For example, the SceneTransition.hideKeyboardOnStart() function uses the doOnStart function to hide the keyboard:

1 override fun transitionFor(previousScene: Scene<*>, newScene: Scene<*>, data: TransitionData?): SceneTransition {
2     return MyTransition()
3         .hideKeyboardOnStart()
4 }

The DefaultSceneTransitionFactory includes this keyboard hiding behavior by default.
